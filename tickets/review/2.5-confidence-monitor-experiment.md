## What was built

Both `monitor.py` and `experiment.py` now emit the shared `confidence.py` `Confidence` band +
`Evidence` block the action plan already uses, so the whole repo speaks ONE confidence vocabulary.
The sample-size discipline that already lived in these modules (monitor's `min_spend` significance
floor + grace; experiment's `min_conversions` gate) was **generalized into the shared `abstain`
verdict, not reinvented**.

### monitor.py
- `classify_ad` gained a `recency_days: int | None = 0` param and now returns a `confidence` dict
  (via `confidence_to_dict`) on every branch:
  - `insufficient` (spend < `min_spend`) → **abstain** (⚪ "insufficient data — too early to judge").
  - protective grace (`watch` for an ad changed within `grace_days`) → **abstain** ("too young to
    judge"). Grace still wins: a young below-floor ad stays `watch`, never `urgent`, and its
    confidence can never read a high-confidence pause.
  - `urgent` / `underperforming` / `ok` → `assess(...)` with `tier=direct_observation`, data band
    from spend/purchases/recency. The row's reasons text is passed as `causal_text`; the descriptive
    reasons contain no causal language so `causal_flag` is False (asserted in tests).
- A small `_abstain_confidence(factors)` helper produces the explicit "decline to judge" verdict for
  the two domain abstentions (significance floor + grace). It is NOT a fabricated score — `abstain`
  is the integer floor of the band scale; grounding is `direct_observation` but the data axis
  abstains so the combined verdict abstains (weaker axis governs).
- `build_watch_report` now passes `recency_days=0` (the window ends at `as_of`, so the data is
  maximally fresh and the call stays clock-free) and attaches both `confidence` (from `classify_ad`)
  and a full `evidence` block (ROAS over the window, spend, purchases, entity=ad, plus a
  `regenerating_query`) to each flagged row.

### experiment.py
- `read_experiment` now adds a structured `confidence` block + an `evidence` block to its output,
  alongside the **unchanged** `verdict` / `caveat` strings (backward compatible):
  - below `min_conversions` per arm → **abstain** (and the existing `INSUFFICIENT DATA` string is
    preserved).
  - `SIGNIFICANT` (p<0.05) with enough data → `tier=ab_experiment`; data band driven by per-arm
    purchases + the p-value → can reach **high**. The experiment is the top grounding tier, so it is
    NOT capped the way a correlational claim is.
  - `NO significant difference` (p≥0.05) → grounding still `ab_experiment` but data band capped at
    **medium** (no proven effect yet); verdict unchanged.
  - Evidence: metric = ROAS lift %, window = experiment window, sample = per-arm purchases (the
    weaker arm governs via `min()`), entity = the experiment's level/ids, with a `regenerating_query`.
    `causal_flag` is False (the A/B IS the causal instrument, so there is no *unsupported* causal
    claim to flag).

## Reviewer focus / known gaps & judgment calls

- **`_abstain_confidence` constructs `Confidence` directly rather than routing through `assess`.**
  This is the one deliberate deviation from "assess is the only scoring path." Rationale: grace ("too
  young") cannot be expressed as a below-floor *sample* (a young ad may have high spend), so there is
  no input that drives `assess` to abstain for it. `abstain` is an explicit refusal-to-score, not a
  free-typed number, so constructing it directly is defensible — but a reviewer may prefer it routed
  through `assess`. Worth a second opinion.
- **Monitor recency is hard-coded to 0.** The watch window always ends at `as_of`, so recency is
  genuinely 0 and the band never rounds down for staleness. This is correct for the watch scanner but
  means the recency axis is effectively inert here. If a future caller scores an older window, it must
  pass a real `recency_days`.
- **`urgent` ads usually read `low` confidence**, because a flagged loser typically has thin
  conversions (< 25 purchases) even with cleared spend. That is honest (the monitor flags for the
  agent to reason over, not auto-pause), but verify it reads as intended and doesn't surprise an
  operator who expects an "urgent" flag to be high-confidence.
- **Experiment `sample_spend=None` + `spend_floor=1.0`.** Spend is deliberately kept off the data
  axis so the conversion floor (`min_conversions`) alone governs abstention. The `1.0` is an arbitrary
  positive sentinel (any value > 0 works since the sample spend is `None`→0). Confirm this reads
  cleanly; an alternative is exposing a named constant.
- **`min()` of per-arm purchases** drives the experiment's data band (the weaker arm governs, matching
  the "both arms ≥ `min_conversions`" gate). Reasonable, but confirm it's the intended sample
  semantics vs. e.g. summed purchases.
- **`high` needs ≥ 100 purchases (4× the conversion floor),** not just ≥ 25 — this is the same
  confidence-core calibration knee the actions ticket documented (its
  `test_action_plan_pause_with_43_purchases_reads_medium_under_calibrated_knee`). The significant test
  uses 100/300 purchases to reach `high`; ≥25 alone yields `medium`. Out of scope to change here.
- **Read-only w.r.t. Meta** preserved — both modules only enrich their output dicts; no new writes.
- No linter/type-checker is configured in the repo, so static checks were not run; tests are the
  floor. The added tests cover the documented use cases but treat them as a starting point.
