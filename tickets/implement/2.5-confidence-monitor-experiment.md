## Why

The parent ticket says the sample-size discipline already lives in `monitor.py` (the `min_spend`
significance floor + protective grace) and `experiment.py` (the `min_conversions` "needs more data"
gate) and should be **generalized, not reinvented**. This ticket has both emit the SAME
`confidence.py` band + abstention verdict the action plan now uses, so the repo has one confidence
language. It also makes the A/B readout — the only top-tier (`ab_experiment`) grounding source —
able to read 🟢 High when significant, which is the visible "confidence rises as grounding improves"
behavior from the parent ticket's use cases.

## What to build

### monitor.py

`classify_ad` already returns `insufficient` below `min_spend` and `urgent`/`underperforming`/
`watch`/`ok` above it. Add a `confidence` block to each flagged row (and the watch report) using
`confidence.assess`:
- `insufficient` classification → map to the `abstain` band (⚪ "insufficient data — keep running").
  This is the existing significance floor, now expressed in the shared vocabulary.
- The protective grace (`watch` for ads changed within `grace_days`) must continue to win — a young
  below-target ad stays `watch`, and its confidence reflects "too young to judge," never `urgent`.
- `urgent`/`underperforming` rows get `grounding_tier == "direct_observation"` and a data band from
  spend/purchases/recency over the window. Pass the row's reasons text as `causal_text` (the watch
  reasons are descriptive, so causal_flag should normally be false — assert it).
- Attach an `Evidence` block (ROAS over the window, spend, purchases, entity = ad) with a
  `regenerating_query` so a flagged ad traces back to a reproducible number.

### experiment.py

`read_experiment` already gates on `min_conversions` (default 25) with an `INSUFFICIENT DATA`
verdict. Add a structured `confidence` block to the readout dict:
- Below `min_conversions` per arm → `abstain` band (keep the existing human verdict string too).
- `SIGNIFICANT` (p<0.05) with enough data → `grounding_tier == "ab_experiment"`, data band driven
  by per-arm purchases + the p-value → can reach **high** (the experiment is the top grounding
  tier, so it is NOT capped the way correlational claims are).
- `NO significant difference` (p≥0.05) → grounding still `ab_experiment` but data band capped at
  medium (no proven effect yet); verdict unchanged.
- Evidence: the comparative metric is ROAS lift; window is the experiment window; sample is per-arm
  purchases; entity is the experiment's level/ids. `causal_flag` is false here because the A/B IS
  the causal instrument — confirm `assess` does not downgrade an `ab_experiment` claim.

Keep the existing `verdict` strings and `caveat` intact (backward compatibility); ADD the
`confidence` block alongside.

## Edge cases & interactions

- **Grace beats pause.** The recently-changed-ad protection must still produce `watch`, never
  `urgent`, and the confidence must not contradict that (no "high confidence: pause" on a protected
  ad).
- **Abstain ≠ low.** Below-floor must be the abstain verdict in both modules, consistent with the
  action plan.
- **Experiment is the only uncapped causal source.** Verify an `ab_experiment` tier with a
  causal-sounding hypothesis is NOT downgraded — the experiment is the causal evidence.
- **Read-only w.r.t. Meta.** Both modules stay read-only; this only enriches their output dicts.
- **Determinism.** Recency is passed as `recency_days` computed from `as_of`/window end, so tests
  stay clock-independent (matches the existing no-`datetime` test style).
- **Backward compatibility.** Keep `verdict`, `reasons`, `caveat`, watchlist shape; only ADD the
  `confidence` (and `evidence`) fields.
